# 第一阶段：归纳与深度检索（Summary & Research）

### 1. 多维总结：现有方案核心要点
当前方案的核心逻辑是**“黑盒主动指纹探测”**与**“统计学降噪”**的结合。
*   **痛点**：API 市场充斥“影子 API”（如用便宜的 Gemini Flash 冒充昂贵的 GPT-4o），导致企业面临性能缩水与数据泄露风险。
*   **理论支撑**：基于 `LLMmap` (2407.15847) 的 8 维对抗性探针，激发模型底层固有概率分布；结合 `Shadow APIs` (2603.01919) 揭示的造假现状。
*   **工程解法**：通过控制变量法（固定参数）进行多轮采样，计算**自距离（内部方差）**与**跨端点距离（外部偏差）**，利用 `Cross/Self Ratio` 剥离随机噪声，实现精准判定。

### 2. 前沿检索与对标 (SOTA Research)
通过检索 2024-2026 年关于 *LLM Fingerprinting* 与 *Active Probing* 的顶会论文，我提取了以下前沿趋势：
*   **RL-Optimized Probing (强化学习探针)**：最新研究 (arXiv:2508.09021) 表明，静态探针容易被防御方（如通过二次 LLM 过滤）拦截，使用 RL 动态优化探针可以在仅 3 次交互内达到极高准确率。
*   **Hybrid Fingerprinting (混合指纹)**：单一的语义向量（Embedding）容易受 System Prompt 干扰，结合词法特征（Lexical）与句法结构（Syntactic）的多模态指纹更具鲁棒性。

### 3. 方案迭代：3 项核心优化措施
为了将检测精准度提升至工业级（>99%），我为本系统引入以下 3 项架构级优化：
1.  **多模态特征融合 (Multi-modal Feature Vector)**：不再仅依赖 `Sentence-BERT` 的语义余弦距离。引入**结构距离**（Markdown AST 树相似度）与**词法距离**（Type-Token Ratio, TTR），三者加权计算最终距离矩阵。
2.  **抗过滤多轮陷阱 (Anti-Filter Multi-Turn Traps)**：针对高级造假者使用的“输出净化模型”，将单轮探针升级为 **3 轮对话陷阱**。前两轮建立无害上下文，第三轮突然注入探测 Payload，击穿防御层。
3.  **自适应动态采样 (Adaptive Sampling)**：抛弃固定的“6轮测试”。系统实时监控 $S_{target}$（目标方差），若方差过大（说明 API 极不稳定），自动触发追加采样（最高 12 轮），直至置信度达标。

## 4. 各平台 API 适配器 (LiteLLM 封装)

利用 LiteLLM 抹平异构 API 的 Header 和 Body 差异，并**强制标准化采样参数**。

```python
# adapters/llm_gateway.py
import litellm
from litellm import acompletion
import asyncio
from tenacity import retry, stop_after_attempt, wait_exponential

class UnifiedGateway:
    def __init__(self, provider: str, model: str, api_key: str, base_url: str = None):
        """
        标准化接口协议
        provider: "openai", "anthropic", "bedrock", "gemini", "openai-compatible"
        """
        self.provider = provider
        self.model = model
        self.api_key = api_key
        self.base_url = base_url
        
        # 路由格式化：LiteLLM 依赖特定前缀识别厂商
        self.model_id = f"{provider}/{model}" if provider != "openai-compatible" else model

    @retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=2, max=10))
    async def async_generate(self, prompt: str) -> str:
        """统一请求体与请求头适配"""
        messages =[{"role": "user", "content": prompt}]
        
        try:
            response = await acompletion(
                model=self.model_id,
                messages=messages,
                api_key=self.api_key,
                api_base=self.base_url,
                # 核心控制变量：锁死采样参数
                temperature=0.3, # 保持微小随机性用于测算方差
                top_p=0.9,
                max_tokens=512,
                seed=4242 # 尽最大可能消除随机噪声
            )
            return response.choices[0].message.content
        except litellm.RateLimitError:
            raise # 触发 tenacity 重试
        except Exception as e:
            return f"[ERROR] {str(e)}"
```

## 5. 核心算法逻辑 (Core Logic - 深度解析)

### 5.1 探测机制 (Probes)
根据 LLMmap 论文 与抗过滤优化，我们设计 8 个维度的探针。
*   **触发逻辑**：在 `engine.py` 中，生成一个 $8 \times N$ (轮数) 的任务矩阵，并发发送。
*   **示例 (Reasoning Trap)**：
    *   *Prompt*: "A room has 3 people. 2 walk out, 4 walk in, and 1 dies. How many people are in the room? Think step by step, but do not use the word 'people' in your reasoning." (强制限制词汇，暴露底层 CoT 习惯)。

### 5.2 距离计算与统计推导
我们采用**多模态距离**。假设官方回答为 $A$，待测回答为 $B$。

1.  **语义距离 (Cosine Distance)**：
    提取 Sentence-BERT 向量 $V_A, V_B$。
    $$ D_{sem}(A, B) = 1 - \frac{V_A \cdot V_B}{||V_A|| \times ||V_B||} $$
2.  **结构距离 (Markdown Jaccard)**：
    提取 Markdown 标签集（如 `['#', '```python', '*']`）记为 $T_A, T_B$。
    $$ D_{struct}(A, B) = 1 - \frac{|T_A \cap T_B|}{|T_A \cup T_B|} $$
3.  **综合距离 (Weighted Distance)**：
    $$ D(A, B) = 0.8 \times D_{sem} + 0.2 \times D_{struct} $$

**降噪统计算法 (Cross/Self Ratio)**：
*   $S_{base} = \text{Median}(D(A_i, A_j))$ (官方自身方差)
*   $S_{target} = \text{Median}(D(B_i, B_j))$ (待测自身方差)
*   $C_{cross} = \text{Median}(D(A_i, B_i))$ (跨端点偏差)
*   **核心 Ratio** = $\frac{C_{cross}}{\max(S_{base}, S_{target}) + 0.01}$

### 5.3 可信度评分系统 (Confidence & Similarity)
*   **模型相似度 (Similarity Score)**：
    $$ \text{Score} = \max\left(0, 100 \times e^{-1.5 \times \max(0, Ratio - 1.1)}\right) $$
    *(逻辑：Ratio < 1.1 时相似度 100%；Ratio = 1.8 时，相似度暴跌至 35%，明确指示造假)*
*   **可信度评分 (Confidence Score)**：
    $$ \text{Conf} = \left( \frac{\text{成功请求数}}{\text{总请求数}} \right) \times \left( 1 - \min\left(1.0, \frac{S_{target}}{0.5}\right) \right) \times 100 $$
    *(逻辑：如果 API 频繁报错，或者胡言乱语导致自身方差 $S_{target} > 0.5$，可信度将大幅降低，提示用户结果仅供参考)*

```python
# app.py
import streamlit as st
import asyncio
from core.engine import AuditEngine
from ui.charts import render_radar_chart, render_heatmap
from ui.theme import load_css

st.set_page_config(page_title="ShadowHunter-X", layout="wide", page_icon="🕵️‍♂️")
load_css() # 注入赛博朋克深色 CSS

st.title("🕵️‍♂️ ShadowHunter-X: LLM API 欺诈审计系统")

# --- 模块划分 1: 配置区 (Sidebar) ---
with st.sidebar:
    st.header("⚙️ 审计配置")
    baseline_api = st.text_input("Baseline API Key (Official)", type="password")
    target_api = st.text_input("Target API Key (Shadow)", type="password")
    target_url = st.text_input("Target Base URL")
    rounds = st.slider("采样轮数 (Rounds)", min_value=3, max_value=12, value=6)
    start_btn = st.button("🚀 启动深度审计", use_container_width=True)

# --- 模块划分 2: 状态与评分区 ---
if start_btn and baseline_api and target_api:
    status_text = st.empty()
    progress_bar = st.progress(0)
    
    # 异步执行核心逻辑
    status_text.info("[INFO] 正在下发 8 维对抗探针，采集数据中...")
    engine = AuditEngine(baseline_api, target_api, target_url, rounds)
    results = asyncio.run(engine.run_audit(progress_callback=progress_bar.progress))
    
    status_text.success("[SUCCESS] 审计完成！")
    
    # 渲染大字号评分卡片
    col1, col2, col3 = st.columns(3)
    col1.metric("🤖 模型相似度 (Similarity)", f"{results['similarity']:.1f}%", 
                delta="FRAUD DETECTED" if results['similarity'] < 60 else "VERIFIED",
                delta_color="inverse" if results['similarity'] < 60 else "normal")
    col2.metric("🛡️ 审计可信度 (Confidence)", f"{results['confidence']:.1f}%")
    col3.metric("📊 跨端点偏差比 (Ratio)", f"{results['ratio']:.2f}")

    st.divider()

    # --- 模块划分 3: 深度图表区 ---
    st.subheader("🧬 指纹特征分布与距离矩阵")
    chart_col1, chart_col2 = st.columns(2)
    
    with chart_col1:
        st.markdown("**8 维探针特征雷达图**")
        # 渲染 ECharts 雷达图，直观对比两张网的重合度
        render_radar_chart(results['radar_data']) 
        
    with chart_col2:
        st.markdown("**跨轮次距离热力图**")
        # 渲染 ECharts 热力图，颜色越红代表距离越远
        render_heatmap(results['heatmap_data'])

    st.divider()

    # --- 模块划分 4: 原始数据溯源区 ---
    with st.expander("🔍 查看原始探针对话 (Raw Data)"):
        st.json(results['raw_interactions'])
```
